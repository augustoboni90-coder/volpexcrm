Flusso operativo consigliato

2A. Telefono pronto

Importa Clienti + Tipo di vendita

Carica un CSV o incolla righe di testo. Se il cliente esiste già nel CRM viene aggiornato solo su telefono, email e indirizzo; tutti gli altri dati rimangono invariati. Il tipo di vendita (contratto, device, riparazione) viene sempre aggiunto come nuovo record.
Regola cliente
Crea se nuovo · Aggiorna solo contatti
Tipi vendita
Contratto · Device · Riparazione
Risultato
—

1. File CSV clienti

Colonne riconosciute (case-insensitive): cf / codice_fiscale, nome, cognome, tipo_cliente, telefono, email, citta, indirizzo, data_nascita. Se il CF esiste, aggiorna solo telefono/email/indirizzo.

2. File CSV tipo di vendita (opzionale)

Il tipo viene rilevato automaticamente dal contenuto. Puoi anche selezionarlo manualmente.

3. File CSV unico (clienti + vendita in un solo file)

Se il tuo CSV ha sia i campi anagrafica sia i campi di un tipo di vendita, caricalo qui: il sistema separa automaticamente le due parti.

Incolla righe da Excel / Google Sheets / testo libero

Incolla una o più righe. La prima riga deve contenere le intestazioni oppure il sistema usa l'ordine: CF, Nome, Cognome, Tipo, Telefono, Email, Città, Indirizzo + campi vendita. Separatori accettati: tab, ;, ,.

Estrai da PDF contratto

Carica uno o più PDF WINDTRE: il CRM prepara automaticamente la stringa completa per il tab Contratti.

Importa archivio dati

Carica un archivio CRM in formato JSON per recuperare dati da backup, migrare da una versione precedente o ripristinare uno storico autorizzato.
Prima dell'import viene creato automaticamente un backup di sicurezza. Dopo l'import controlla i dati e usa CRM → Google per sincronizzare il foglio.
Protocollo collegato a questa Web App: GET action=ping, GET action=list, POST action=pushAll. La sincronizzazione CRM → Google usa un invio ottimizzato. Da qui puoi anche importare un backup JSON del vecchio CRM.
